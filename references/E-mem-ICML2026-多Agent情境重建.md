# E-mem: Multi-agent based Episodic Context Reconstruction for LLM Agent Memory

**arXiv:2601.21714 (v5) | ICML 2026**
作者: Kaixiang Wang, Yidan Lin, Zihan Wang, Bunyod Suvonov 等
代码: https://github.com/dog-last/E-mem

---

## 核心思想

> **记忆不是"检索"出来的，而是"重建"出来的。**
> 所有的记忆预处理（压缩成 embedding / 知识图 / 摘要）都在破坏因果链。

**当前问题**：现有方法都在做"记忆预处理"——把原始对话压缩成固定结构（embedding、知识图、摘要）。这个压缩过程本身破坏了时间顺序和上下文依赖。

**核心创新**：不做记忆压缩，保留完整原始上下文，用多 Agent 架构在原始上下文中"重建"记忆。

---

## 与 MRAgent 的核心差异

| 维度 | MRAgent | E-mem |
|:---|:---|:---|
| 问题诊断 | 检索范式不行（被动→主动） | 记忆压缩不行（preprocessing 破坏了上下文）|
| 记忆形式 | Cue-Tag-Content 图 | 完整未压缩原始文本块 |
| 检索主体 | LLM 直接遍历图 | Router + 助理 SLM |
| 推理位置 | 图遍历时用 LLM 选路径 | 助理在自己的完整上下文中推理 |
| 小模型利用 | 不需要 | **核心设计**（助理用 SLM） |
| 时间追踪 | 通过 tag 隐式 | 通过时间戳显式（关键！）|

## 框架架构

### Heterogeneous Hierarchical Master-Assistant

```
             ┌──────────────────────┐
             │   Master Agent        │   ← 大模型 (全局规划+证据合成)
             └──────────┬───────────┘
                        │ 激活指令
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
    ┌──────────┐ ┌──────────┐ ┌──────────┐
    │ Asst 1   │ │ Asst 2   │ │ Asst 3   │ ← SLM (小模型)
    │ Episode1 │ │ Episode2 │ │ Episode3 │   保持完整原始文本
    │ 摘要1    │ │ 摘要2    │ │ 摘要3    │   轻量摘要(路由用)
    └──────────┘ └──────────┘ └──────────┘
```

### 内存构建

- 滑动窗口分割原文，带重叠（保留边界连续性）
- 每个窗口由一位助理管理，保存完整未压缩文本
- 支持 O(1) 流式更新（追加到当前活跃窗口）
- 窗口满 → 固化 → 新助理实例化 + 继承重叠区域

### 三段式路由（Multi-Pathway Routing）

三段路径并行搜索，取并集：

1. **Global Alignment（全局对齐）**
   - 用预计算摘要做向量+稀疏匹配
   - 粗粒度语义过滤
2. **Semantic Association（语义关联）**
   - 对原始文本块做向量相似度匹配
   - 补偿摘要可能丢失的细节
3. **Symbolic Trigger（符号触发）**
   - BM25 关键词精确匹配
   - 确保实体级精确召回（特定名字、ID 等）

### 推理过程

1. **路由**定位相关助理 → **并行激活**
2. 每个助理在**自己的完整上下文中**主动推理（不是提取文本片段，而是理解并生成回答）
3. 输出带**绝对时间戳**的证据（关键！时间线对齐）
4. **Master Agent 聚合**：
   - **Direct Inference**：直接合并证据
   - **Iterative Reasoning**：不足以回答则生成子查询，继续迭代

### 时间戳机制（区别于 MRAgent 的关键）

每个证据带有绝对时间戳 τ，Master Agent 在处理时用时间线解决冲突：
- 同一个物体在不同时间处于不同位置 → 取最新时间戳
- 事件的先后顺序 → 按时间线重新排列

## 局限

- 助理 SLM 实例数线性增长
- 窗口大小 L 影响性能
- 没有冲突处理的正式机制
- 部署复杂度高于传统的单模型方案

---

## 启发

1. **多 Agent 记忆架构**：4070 + Qwen2.5-14B 做助理，大模型 API 做主控
2. **三路路由**：语义 + 向量 + 关键词 并行检索（比单一策略鲁棒）
3. **时间戳显式化**：任何记忆操作带时间，冲突按时间排序
4. **助理本地推理**：在小模型本地完整上下文做推理，减少大模型调用
5. **与 MRAgent 互补**：E-mem 解决了 MRAgent 不处理的上下文破坏问题，MRAgent 解决了 E-mem 检索精度不够的问题
